Notion content tokens

In the Notion project body, write tokens as a single paragraph block exactly as shown. The sync script (scripts/sync-notion.mjs) detects these and rewrites them as renderable blocks in data/projects.json. Files always live in images/projects/<slug>/.

Single image — {{img:filename}}

{{img:cover.jpg}}
{{img:hero.jpg|Caption shown below the image}}

Renders full-width with 0 margin against the next media block. Caption is optional after a |.

Video — {{video:filename}}

{{video:intro.mp4}}

Renders muted + looping, autoplays only when on screen (IntersectionObserver). Same caption syntax with |.

Image slider — {{slider:files…}}

{{slider:03.jpg|04.jpg|05.jpg|06.jpg|07.jpg}}
{{slider:03.jpg|04.jpg|05.jpg|autoplay=true}}
{{slider:03.jpg|04.jpg|05.jpg|caption=Brand artifacts|autoplay=true}}

Pipe-separated filenames render as a horizontal scroll-snap carousel with pagination dots. Trailing key=value entries set slider options instead of frames. Supported options:

  • autoplay=true — opt in to auto-advance every 2.5s. Off by default; the slider only cycles on its own when this is set.
  • caption=... — overall caption shown under the slider.
  • style=fade — reserved for cross-fade transition (slide is the default).

Auto-play behavior (when enabled):

  • Ease-in-out cubic 900ms tween (custom RAF, not native smooth-scroll — keeps timing consistent across devices)
  • Pauses when scrolled out of view (IntersectionObserver, threshold 0.4)
  • Clicking a dot stops auto permanently — user takes manual control
  • Swipe / trackpad scroll cancels the in-flight tween so finger input always wins
  • Honors prefers-reduced-motion — no auto cycle if the visitor opted out

UI exploration variants live at slider-demo.html (V1–V4 + V3a–V3d auto-advance flavors).

Display letterform — {{letter:Nū}}

{{letter:Nū}}
{{letter:Fik}}

Single oversized letterform as a typographic break in the body. Uses acumin-pro-condensed at clamp(60px, 11vw, 140px), mix-blend soft-light on the page tone so it reads as embossed.

Rotated photo cluster — {{polaroids:files…}}

{{polaroids:a.jpg@-12,b.jpg@0,c.jpg@8}}

Comma-separated filenames, each optionally followed by @<degrees> for rotation (positive or negative, decimals OK). Cards render centered in a row, wrap on narrow viewports. Use this for scattered shots that should feel hand-placed rather than gridded.

Side-by-side mockup pair — {{pair:a.jpg|b.jpg}}

{{pair:333_biz_mockup3.jpg|333_mockup_02.jpg}}

Exactly two pipe-separated images. Renders edge-to-edge across the viewport (breaks out of the body column) at a 1400:933 (~3:2) aspect ratio with 29px gap. Use for paired comparison shots.

Full-bleed stack — {{stack:files…|indent=…}}

{{stack:a.jpg|b.jpg|c.jpg|d.jpg}}
{{stack:a.jpg|b.jpg|c.jpg|d.jpg|indent=1,2}}

Pipe-separated images stack vertically edge-to-edge at 2123:1415 (~3:2) with 40px gap. indent= takes a comma-separated list of 0-based frame indices that should sit inset 40px each side (use it to break visual rhythm — usually inner frames).

Per-project page tone — Notion DB "Tone" property

Add an optional rich text property named Tone to the Notion Projects DB. Paste a single hex color (e.g. #906e47) and the project detail page tints its body background to that color (denim texture + soft-light blend ride on top). Leave empty for the default #E7E7E7 page tone. Invalid values are skipped with a warning in the sync log.

Adding the image files

  1. Drop the assets in images/projects/<slug>/ with the exact filenames you referenced in Notion
  2. Compress >2MB JPGs (target ≤1MB) — large hero images are fine but body images don't need print resolution
  3. Commit + push the images to v2
  4. Trigger "Sync Notion → projects.json" on GitHub Actions (Actions tab → Run workflow → branch v2) so data/projects.json picks up the new tokens

Project rules

  • No italic, ever — global rule. Don't use font-style: italic, <em> for italic effect, or <i>.
  • Notion → JSON sync — projects come from the Notion DB via scripts/sync-notion.mjs. Run with NOTION_TOKEN=xxx node scripts/sync-notion.mjs; it writes data/projects.json.
  • .DS_Store — macOS metadata. Always excluded from commits even when it shows up dirty in git status.
