Quick start

새 프로젝트 추가하는 가장 짧은 흐름:

  1. 노션 Portfolio DB 에 새 row 추가, 필수 칼럼 채우기 (Name / Slug / Published / Order / Cover / Category / Year / Description / Tags)
  2. (선택) Tone / Tone Text 에 hex 색상 입력 — 프로젝트 페이지의 배경/텍스트 색상
  3. 본문에 토큰으로 미디어 배치: {{slider:a.jpg|b.jpg|autoplay=true|style=marquee}} 등
  4. 이미지 파일은 로컬 repo 의 images/projects/<slug>/ 폴더에 같은 이름으로 넣고 commit + push
  5. GitHub Actions 에서 "Sync Notion → projects.json" 트리거 → 1–2 분 뒤 도메인 반영

Projects DB 칼럼

노션 DB 의 각 칼럼이 사이트의 어디로 들어가는지:

칼럼 타입 역할
Name Title 프로젝트 이름 — 메인 갤러리, 상세페이지 헤더 타이틀 (예: "333")
Slug Rich text URL 식별자. 소문자 + 숫자 + 하이픈만 (^[a-z0-9-]+$). 폴더명도 동일. 예: 333, proof-of-retreat
Published Checkbox 체크돼야 sync 가 가져감. 체크 안 된 row 는 사이트에 안 나옴 — 임시 작업용으로 활용
Order Number 갤러리 정렬 순서 (오름차순). 작을수록 먼저
Cover Rich text 커버 이미지 파일명 (확장자 포함). images/projects/<slug>/<cover> 경로로 해석됨. 예: 333_01.jpg
Category Select 예: Brand Identity, Editorial, Packaging
Year Rich text 예: 2026
Client Rich text (JSON 에는 포함되지만 페이지에 직접 표시되진 않음 — 향후 노출 가능)
Description Rich text 상세페이지 헤더의 짧은 소개 (Special Elite 12px)
Tags Multi-select 예: Branding, Package, Spatial. 메인 갤러리 메타에 표시
Tone Rich text (hex) (선택) 페이지 배경색 hex. 예: #906e47. 비우면 기본 #E7E7E7
Tone Text Rich text (hex) (선택) 모든 텍스트 색상 hex. 예: #45300F. 비우면 기본 #090909 / #7b7b7b

Slug 주의: 한 번 정하면 URL 이 그 값으로 고정됨. 나중에 바꾸면 외부 링크가 깨짐 + 이미지 폴더명도 같이 옮겨야 함. 처음부터 신중히.

Tone — 프로젝트별 색상

각 프로젝트가 자기 브랜드 컬러를 가질 수 있게 2개 hex 칼럼만 채우면 됨.

Tone (배경)

Tone 에 hex 입력 → 페이지의 --ink CSS 변수가 그 색으로 override. 데님 텍스처와 dot grid 가 위에 soft-light 로 블렌드.

Tone:       #906e47    ← 333 브랜드 brown
            #1a1a1a    ← 어두운 톤
            #f5e8d8    ← 따뜻한 ivory

Tone Text (텍스트)

Tone Text 에 hex 입력 → 페이지의 모든 텍스트 (.proj-title, .pb-h1/2/3, .pb-paragraph, .proj-meta-*, .back-btn) 가 그 색으로 통일.

Tone Text:  #45300F    ← brown bg 에 dark brown 텍스트 (333 톤온톤)
            #ffffff    ← 어두운 bg 에 흰 텍스트
            (비움)     ← 기본 #090909 / #7b7b7b

두 칼럼은 완전 선택. 둘 다 비우면 기본 톤 (Denim Gray) 으로 렌더. 한쪽만 설정해도 됨.

본문 토큰

노션 본문의 paragraph 블록 한 줄에 토큰만 적으면 sync 가 그걸 인식해서 해당 미디어 블록으로 변환. 토큰 형식 외에는 일반 paragraph / heading / bulleted list / quote 등 노션 기본 블록 그대로 사용.

중요: 토큰은 혼자 있는 paragraph 여야 함. 다른 텍스트와 같은 줄에 섞으면 인식 안 됨.

슬라이더 vs polaroids / pair / stack — 모두 이미지 여러 장을 다루지만 축이 다름. 슬라이더는 한 번에 한 장만 보이고 시간으로 넘어감 (시간 축). polaroids / pair / stack은 여러 장이 동시에 한 화면에 펼쳐짐 (공간 축).

용도 차이: polaroids = 캐주얼한 사진 모음 손맛, pair = 두 장 직접 비교, stack = 시퀀스/카탈로그처럼 죽 나열.

이미지 파일 워크플로우

파일 경로 규칙

images/projects/<slug>/<filename>
        예: images/projects/333/333_01.jpg
            images/projects/proof-of-retreat/por_01.jpg

토큰에 적은 파일명이 그 폴더 안에 실제로 있어야 사이트에서 보임. sync 는 파일 자체를 다운로드하지 않음 — 경로 문자열만 적어둠.

권장 사이즈

  • Cover: 1200×1200 정사각형 (메인 갤러리에서 1:1 crop 됨), 800KB ~ 1.5MB
  • 본문 이미지: 긴 변 최대 2400px, JPG quality 70~80, 파일당 500KB ~ 1.5MB
  • 비디오: 1080p mp4, h.264, 2MB 이내 권장

압축 권장

원본 8MB+ JPG 그대로 푸시하면 페이지 로딩이 느려짐. macOS 에선 sips -Z 2400 -s formatOptions 70 input.jpg --out output.jpg 한 줄로 ~800KB 까지 줄어듬.

커밋 & 푸시

git add images/projects/<slug>/*
git commit -m "Add <slug> cover and body images"
git push origin v2

동기화 트리거

노션 변경 사항은 자동으로 즉시 반영 안 됨 — GitHub Actions 워크플로우가 돌아야 함.

자동 (6시간마다)

cron 으로 6시간마다 자동 실행. 급하지 않으면 그냥 기다림.

수동 트리거

  1. GitHub repo → Actions 탭
  2. 좌측에서 "Sync Notion → projects.json" 선택
  3. 오른쪽 "Run workflow" 버튼 → Branch: v2 → Run
  4. 1–2분 뒤 완료 → data/projects.json 자동 commit + Vercel 자동 배포

변경 사항 확인

sync 결과는 data/projects.json 의 새 commit (작성자 github-actions[bot]) 로 확인. 도메인 (nu-fik.com) 은 그 commit 푸시 직후 자동 재배포됨.

자주 하는 실수

  • 토큰을 텍스트 중간에 섞음 → paragraph 블록 한 줄에 토큰만 있어야 인식됨. "여기 이미지 → {{img:...}} ← 보여" 처럼 쓰면 그냥 문자열로 남음
  • 이미지 파일을 잊고 노션만 업데이트 → JSON 에는 경로 들어가지만 사이트에서 broken image. 노션 + 이미지 파일 + sync 트리거 3종 세트 항상 같이
  • Published 체크 누락 → sync 는 Published: true 만 가져옴. 임시 row 만들 땐 unchecked 로 두면 안전
  • Slug 에 대문자/공백/특수문자 → sync 가 skip 함 (^[a-z0-9-]+$ 만 허용). "Cha Cha Cha" → cha-cha-cha
  • 파일명 대소문자 오타 → 토큰 안 파일명과 폴더의 실제 파일명이 정확히 일치해야 함. Mac 에선 대소문자 무시되지만 Linux 서버에선 구분됨
  • 슬라이더에 파일 1개만 → 슬라이드 안 됨. 2개 이상 필요. 1개면 그냥 {{img:...}}
  • Tone 에 색이름 ("red", "blue") → hex 만 허용. #ff0000 처럼 적어야 함
